부동산 DB 규칙 (규칙 페이지)

1 주소 기반의 데이터베이스
2 핵심키는 별도 고유 ID — 주소는 속성일 뿐 (같은 주소라도 층수/호실 다르면独立된 데이터)
3 증가는 가능, 삭제는 불가능 (8번 참조)
4 부동산 특성에 따라 테이블값 형식 특정
5 데이터베이스 수정 가능
6 동일 주소 + 동일 층수/호실 조합에서만 중복 불가 — 다른 층수/호실은 같은주소라도 독립된 데이터로 허용
7 데이터베이스를 직접 만들거나 수정하는 특징이 있으니 로그 기록 필수
8 수정 시 새 데이터 등록 + 기존 같은 주소+층수/호실 데이터 비활성화 — 삭제는 불가능

3대 데이터베이스 연동 구조

환경: IndexedDB (온라인/오프라인 무관)
3개 DB를 하나의 데이터처럼 다루며, 앱 로직에서 ID로 자동 연결됩니다.
CSV 파일 export/import 기능으로 백업·이식 가능.

9 환경: IndexedDB(브라우저 내장), CSV export/import 병행 — 모든 브라우저에서 오프라인 동작
10 DB 간 연결: 모든 테이블에 `id`(고유키) + `property_id`(외래키) — 앱 로직에서 한 번에 조인

계약자 식별정보 처리 규칙

11 ID 정보: `id_type`(주민등록/여권/사업자 등록) + `id_value`(번호) 별도 컬럼으로 저장 — 키값은 별도 고유 ID이므로 중첩 불가
12 주민등록번호 뒷자리 DB 절대 보관 금지 — 앞 6자리(생년월일)만, 실제 전체 확인시 신분증사본 이미지 파일만 열람
13 신분증 사본은 이미지/PDF 파일로 별도 폴더 저장, DB에는 URL 경로만 참조

3대 데이터베이스 테이블 구조

① 계약자 데이터 (tenants)

필드타입설명
idINTEGER PK계약자 고유 ID (연결 핵심)
nameTEXT성명
phoneTEXT연락처
addressTEXT거주지 주소
id_typeTEXT신분증유형: 주민등록/여권/사업자
id_valueTEXT식별번호(주민: 앞6자리만, 여권: 평문)
id_doc_urlTEXT신분증 사본 파일 경로 (업로드 경로 기준)
typeTEXT CHECK임대인/임차인
contract_idsTEXT(JSON)관련 계약서 ID 목록 (예: [1,3])

② 계약서 데이터 (contracts)

필드타입설명
idINTEGER PK계약서 고유 ID (연결 핵심)
property_idINTEGER FK → properties.id대상 부동산 매핑
tenant_idsTEXT(JSON)관련契約자 ID 목록 (예: [2,5])
start_dateDATE계약 시작일
end_dateDATE계약 종료일
deposit_wonBIGINT보증금 (원)
monthly_rent_wonBIGINT월 임대료 (원)
statusTEXT활성/해지/갱신 등

③ 관리비 및 청구 데이터 (bills)

필드타입설명
idINTEGER PK청구서 고유 ID
property_idINTEGER FK → properties.id대상 부동산
contract_idINTEGER FK → contracts.id관련 계약서 (임대인과 연결)
tenant_idsTEXT(JSON)실제 납부자 계약자 ID
bill_typeTEXT CHECK재산세/전기/수도/통신 등
amount_wonBIGINT청구 금액 (원)
issued_dateDATE고지일
due_dateDATE납부기한
paidINTEGER 0/1결제 여부

데이터베이스 간 조인 예시

부동산 → 계약자 → 계약서 → 관리비
properties.id = contracts.property_id부동산 ↔ 계약서 매핑
contracts.contract_ids (JSON) → tenants.id계약서 ↔契約자 연결고리
bills.tenant_ids (JSON) → tenants.id관리비 ← 契約자 (실제 납부자)
bills.contract_id → contracts.id관리비 ↔ 계약서 직결 연결고리

기존 부동산 테이블 (properties)

필드타입설명
idINTEGER PK고유 키값 (별도)
addressTEXT주소 (속성, 중복 허용)
floor_numINTEGER층수 (중복判定용)
room_numINTEGER호실 (중복判定용)
property_typeTEXT CHECK상가/오피스텔/아파트/토지
area_sqmREAL면적 (m²)
detailsTEXT추가 정보
versionINTEGER버전 관리
is_activeINTEGER 0/1활성/비활성
