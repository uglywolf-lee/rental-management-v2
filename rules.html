📋 부동산 관리 시스템 — 규칙 + 필드표 + 인터페이스 완전 연동본 (최종)

📜 핵심 DB 규칙 (1~8)

1주소 기반 데이터베이스 (부동산의 근원은 주소지)
2핵심키는 별도 고유 ID — 주소는 속성일 뿐
3증가는 가능, 삭제는 불가능 (영구 폐기 절대 금지)
4부동산 특성에 따라 테이블값 형식特定 (상가/오피스텔/아파트/토지 등)
5데이터베이스 수정 가능
6동일 주소 + 동일 층수/호실 조합에서만 중복 불가
7DB 직접 조작 시 로그 기록 필수
8수정 시 신규 INSERT + 기존 비활성화 (is_active=0, DELETE 금지)

📋 시스템 필드 전역 표준 (FIELDS.md 기준 — A~I 인터페이스 매핑됨)

9시간/날짜: DB저장: YYYY-MM-DD HH:MM:SS + 부동산 화면출력은 오직 'YYYY-MM-DD (요일)'
10전화번호: 반드시 NN-NNNN-NNNN 형식 고정 (하이픈 포함 14자리. 공백/괄호 금지)
11이메일: lowercase_text@domain.tld (소문자 강제 변환 저장)
12금액(DB저장): BigInt 원단위 정수. (예: 5000000)
13금액(화면표시): 공과금 외 천/만원 단위, 공과금은 원단위 고정
14KST 한국타임: 모든 테이블의 timestamps 통일. 부동산 항목 화면에서 시간 배제

👥 사용자 권한 + 감사로그 + 시스템 백업 (15~23)

15관리자(거의 전권): 모든 데이터 조작 권한 / 불특정사용자 접근차단
16사무인원: 검색 + 입력만 (수정/삭제 불가)
17건물유지보수인: 시설만 입력+조회 / 계약서수정불가
18누가 어디에 무슨 짓을 했는지: 로그파일 관리 + 감사로그테이블(db_logs) 필수저장

💾 시스템 백업 및 롤백 규칙 (19~23)

19system_snapshots: 수정전 반드시 이전상태 JSON 자동저장 (target_table_name, target_id, data_snapshot_json)
20rollback(복구): 실수시 data_snapshot_json 값을 찾으면 INSERT/UPDATE — 즉시 복구 가능
21users + role_name: super_admin / office_worker / maintenance_staff (아이디/이메일/비밀번호해시)
22audit_logs.detail_log_json: 변경전후값 기록 ('{ "target_id": 45, "old_value": 500000, "new_value": 600000 }')
23핵심 데이터(contacts/bills/contracts) 수정전 ALWAYS 롤백-스냅샷 → is_restored=1 마킹 필수

🖥️ 구현할 인터페이스 목록 (A~I — 필드명 FIELDS.md 완전일치)

A
A. 부동산 관리:
입력: buildings.name/address/floors/rooms_count + rooms.floor_no/room_no/area_sqm(FIELDS.md①②일치매핑)
수정: is_active=0→신규ROW(8번규칙매핑) / 검색: address LIKE '%%'주소별현황리스트
B
B. 계약서 관리:
입력: contracts.lease_type/deposit_amount/monthly_rent/commission_fee(중개수수료)+owner/tenant_contact_id(contacts.id매핑)
계약서원본PDF/JPG→uploads/contracts/2025_001.pdf (상시옆에 떠있는 visible 저장)
수정: 신규INSERT 기존is_active=0(8번규칙+시스템백업스냅샷자동저장) 매핑
C
C. 계약자 관리:
입력: contacts.company_or_name (필드명FIXED), representative_name대표자명(FIELDS③매핑), contact_info='NN-NNNN-NNNN', email, documents_json
수정: 동일id기저is_active=0 +신규ROW / 검색: company_or_name LIKE '%%'/연체조회/해지여부일관매핑
D
D. 공과금 관리:
입력: bills.bill_type(='전기요금'/'관리비'/...)/amount(원단위정수저장/FIELDS⑤매핑)/due_date
수정: is_paid=1/금액변경 / 검색: 미납현황(bills.is_paid=0)/물별청구리스트
E
E. 월세 납입 관리:
입력: deposit_amount + monthly_rent(bills.amount)/납부일/정산내역(보증금반환)
수정: 납부이력변경 연체처리이역 / 검색: 월별납현황총계요약/미납자리스트
F
F. 시설유지보수(INCIDENTS):
입력: incidents.description(room_id FK+reported_by_id(FK→users.id) 매핑필드)/ estimated_cost/estimated_cost
수정: status('접수중'→'완료') / 실제수리비정산 / 검색: 부동산별유지이력(연결고리핵심)
G
G. 통합 월세 관리 인터페이스:
전체: 부동산별·계약자별월세총입금+모든공과금( bills.amount )/보증금반환현황판한눈에매핑필드
H
H. 통합 계약 관리 인터페이스:
전체: 모든계약의만기/갱신/해지현황 + 다음행위추적표 — 계약서+계약자+부동산연결고리 한눈에
I
I. 직원용 대시보드 (전체종합):
부동산총수량·계약현황·미납월세(contacts.bills.is_paid+ bills.amount연결고리)/예외사건(파손/repairs) 통합.
users.role_name('super_admin'/'office_worker'/'maintenance_staff')로접근제한 — audit_logs.detail_log_json전부매핑
