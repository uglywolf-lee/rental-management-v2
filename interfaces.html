📋 부동산 관리 시스템 인터페이스 목록 (구현 우선순위)

A. 부동산 관리

입력부동산 건물 및 인프라 호실 등록 buildings.name / .address / rooms.room_no / .area_sqm
수정현황 변경 및 면적/특성 수정 시 is_active=0 비활성화 후 신규 Row 복제 INSERT (규칙 8 준수)
검색주소/유형별 현황 리스트 조회 address LIKE '%주소%'

B. 계약서 관리

입력계약 조건 기입 contracts.lease_type / .deposit_amount / .monthly_rent / .commission_fee
수정계약 변경 및 갱신 시 자동 스냅샷 JSON 백업 생성 후 INSERT 버전 관리 (규칙 19~23)
연동증빙 서류(PDF/JPG) 업로드 디렉토리 바인딩 및 화면 내 상시 고정형 뷰어 UI 노출

C. 계약자 관리

입력인적사항 통합 입력 contacts.company_or_name (상호/성명), representative_name (대표자명 필수)
수정연락처 기입 칸 하이픈 포함 14자리 고정 규격 유효성 검증 레이어 강제 적용 contact_info ('NN-NNNN-NNNN')
검색계약자 리스트 및 관계자 카테고리별 B2B 마스터 데이터 연체/해지 현황판 출력

D. 관리공과금 및 통합 검침

입력검침등록 및 고지서 발급 양방향 멀티 입력 UI 구현 meter_readings.current_cumulative_reading (당월 누적값)
로직당월 검침 기입 즉시 전월 대조 사용량(usage_amount) 연산 및 최종 확정 요금 bills.amount 실시간 자동 파생 생성
검색미납 현황 및 호실별 청구 내역 요약 리스트 조회

E. 월세 납부관리

입력수납일, 정산 입금 내역 및 퇴거 시 보증금 반환 정산 처리
수정미납 계좌 확인 시 수납 상태 여부 필드 실시간 반전 처리 및 연체 기록 동기화 bills.is_paid (0/1)
검색월별 수납 현황 총계 요약 대시보드 및 연체자 미납 리스트 자동 추출

F. 시설유지보수 (INCIDENTS)

입력파손/훼손 내용 기입 incidents.description, 유지보수인 계정 ID 매핑 reported_by_id, 수리 추산액 estimated_cost 등록
수정진행 처리 상태값 실시간 변경 status ('접수중' → '보급완료') 및 실제 정산 수리비의 임차인 청구 연계
검색부동산 현장별 시설 유지보수 이력 판넬 및 사건사고 정산 추적

G/H/I. 통합 수납·계약 실무 대시보드 (종합)

전체부동산별·계약자별 월세 총수입 판넬, 임대차 만기/갱신 현황 시각화 및 다음 행위 추적 테이블 통합 출력
메모✍️ 실무 진행 메모창 <input class="ledger-memo-input"> 배치 → 변경 시 onChange 이벤트 기반 memos 테이블 자동 영속화 (규칙 35)
격리로그인 계정의 권한 등급 users.role_name에 따른 UI 대시보드 레이아웃 접근 제한 및 데이터 노출 범위 강제 격리

★ J. 팀원 관리 인터페이스 (team_management.html)

제약기존 구형 이메일 계정 입력 칸 전면 폐기, 사번 기반 고용 번호 식별자 고정 매핑 users.employee_no ('EMP-001') (규칙 40)
검증접속 비밀번호 생성 시 해시 연산 전 최소 길이 6자 제약 조건 유효성 검증 로직 강제 적용 password_hash
→ 사내 임직원 권한 제어 및 신규 직원 등록 계정 관리 화면
💡 참고: 모든 화면 UI 내부의 placeholder, 라벨, 테이블 헤더에는 영문 필드명 표기가 엄격히 제한되며 오직 우리말로 치환됩니다 (규칙 26).
