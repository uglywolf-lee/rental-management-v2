📋 부동산 관리 시스템 인터페이스 목록 (구현 우선순위)

A. 부동산 관리

입력부동산 등록 주소/층수/호실/유형(상가등)/면적
수정현황변경 면적확장 특성변경 (삭제는 없음)
검색주소/유형별 현황리스트

B. 계약서 관리

입력계약조건 (보증금/임대료/기간/전월세 형태)
수정조건변경 (이력관리 + 기존 데이터 비활성화)
검색부동산별 계약현황 / 기간별 갱신표

C. 계약자 관리

입력성명/연락처/신분증유형+번호/거주지
수정개인정보변경 이력 (삭제는 없음)
검색계약자리스트 / 계약현황/연체/해지여부

D. 관리공과금 (관리비)

입력재산세/전기/수도/난방/통신 고지금액/납부기한
수정금액변경/납부확인 (이력관리)
검색미납현황/물별 청구리스트

E. 월세 납부관리

입력납부일/금액/정산내역 (보증금 반환등)
수정납부이력변경 연체처리 이역 (삭제는 없음)
검색월별 납현황/총계 요약 / 미납자리스트

F. 시설유지보수 (INCIDENTS)

입력파손/훼손신고 repair필요사항/비용추산
수정처리상태변경 (접수중→완료) / 실제수리비정산
검색부동산별유지이력 / 파손별비용추계

G. 통합 월세 관리 인터페이스

전체부동산별/계약자별 월세총수입/미납목록/연체일수 통합현황판
→ 월세+공과금+보증금반환을 한눈에

H. 통합 계약 관리 인터페이스

전체모든계약의만기/갱신/해지현황 / 다음행위추적표
→ 계약서+계약자+부동산 연결고리 한눈에

★ I. 직원용 대시보드 (전체종합)

조회부동산총수/계약현황/미납월세/예외사건(파손등) 통합현황
→ 모든 데이터를 요약하고 한눈에 확인하는 직원용 메인화면
💡 참고: 각 항목(입력/수정)은 별도로 구현됨, 검색은 조회용 별도 페이지
인터페이스는 실제 데이터베이스 생성 전에 먼저 설계 (DB는 채워넣어야 완성)
